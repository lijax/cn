    HTML5之互联网的未来

    HTML5之互联网的未来

    最近对HTML5非常感兴趣，发现它才是互联网的未来，能够解决所有Flash和微软正大力开发推广的Silverlight所能解决的问题。连3D显示都能在网页中实现，包括网页交互，渲染图片，其炫目不亚于flash。最大的优势就在于它只是一个协议，所有的炫目效果都是网页的整体而不是任何的插件。我觉得Flash和Silverlight很可能因为它的存在和发展而最终淘汰。或许因为不太喜欢flash的菜单及讨厌的下载工具对它的识别，总是对flash有几分不满。看见HTML5的效果之后我欣喜若狂。

    软件产业最终将会走向互联网平台（当然移动平台还说不准），连3D的大型游戏都能在浏览器中实现了，还有什么软件功能不能在互联网实现吗？所以桌面级的应用可能都会被互联网的应用迅速替代，桌面平台最终会被淘汰。Google对未来互联网和信息产业的布局非常深远，其布局的操作系统即是网络操作系统，只支持互联网应用，可以想象引领互联网科技的趋势。而其所谓云计算，最终的目的就是用互联网应用解决所有问题，包括桌面平台所能解决的问题。

    互联网应用有很多好处，方便性：随时随地，只要有一个支持连接互联网的浏览器就行，不管是任何平台（也有可能未来所有平台都只支持互联网）；安全性：不用考虑病毒木马的存在，那是服务提供商所要考虑的问题；实时交互性：随时分享所有信息，文件从创建到使用都在云端，免去上传下载，要想给别人只需要一个简单的共享。

    据说HTML5会在2012年对外正式发布，正式应用于网络，期待那一天的到来，期待未来的互联网世界。为积极参与，现在开始学HTML5。（感新鲜看一下导航里面的HTML5演示，超级酷）。

    Linux和windows引导修复方法

    装ylmf系统的过程中不小心把系统引导装在了window系统所在的硬盘，本来是想装在移动硬盘上，这样方便即插即用的。现在到好，不插移动硬盘连window都启动不了啦，移动硬盘上的系统在别的机器上也没法引导。

    找了一下解决办法，分两步。

    • 首先将grub引导写入Linux所在移动硬盘的mbr：

    sudo grub-install /dev/sdb（这里sdb是我对移动硬盘，根据情况而定）

    • 再对windows所在的系统进行修复，使用ms-sys工具:

    ms-sys不能通过软件库在线安装，需要单独下载，可通过http://ms-sys.sourceforge.net/下载。

    在这之前确保你已经安装过gettext软件包，否则安装过程中可能会报 msgfmt：命令未找到 错误。

    sudo apt-get install gettext

    按照上面链接中的安装步骤进行ms-sys安装。

    安装完，执行下面的命令修复windows引导

    sudo ms-sys -m /dev/sda（sda是我的windows所在硬盘）

    今天想修改默认的文本编辑器gedit为sublime_text,更改了几次都没成功，原来关联文件一共有三个，记录以备用。

    /etc/gnome/default.list or /usr/share/applications/default.list  
~/.local/share/applications/mimeapps.list  
~/.local/share/applications/default.list

    用文本编辑器更改上述三个文件中的信息，替换gedit.desktop为sublime-text-2.desktop（确认/usr/share/applications/下的名称）
